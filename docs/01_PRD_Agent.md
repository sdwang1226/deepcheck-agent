## 一、产品背景与定位

### 1.1 为什么要做这个 Agent

个人投资者做投研决策时面临三个痛点：

| 痛点 | 现状 | Agent 解法 |
|------|------|-----------|
| 数据分散 | 行情看一个 App，财报查一个网站，新闻刷另一个 | 一次对话，Agent 自动跨数据源聚合 |
| 认知门槛高 | PE/PB/ROE 指标需要专业知识才能正确解读 | Agent 用自然语言解释指标含义 + 横向对比 |
| 信息过载 | 10-K 年报 200+ 页，核心风险埋在第 30 页 | RAG 检索 + LLM 摘要，10 秒出结论 |

### 1.2 产品定位

> DeepCheck 是一个面向个人投资者的 AI 投研助手。通过自然语言对话，自动调用实时行情 API、财报 API 和 RAG 文档检索，输出带来源引用的投研简报。

### 1.3 生态位与对标分析

DeepCheck 构建在长桥 OpenAPI 之上，与长桥自有 AI 产品是**"平台 + 应用"**的关系，而非直接竞品。对标分析需从生态位角度理解：

**生态位关系：**

| 关系类型 | 产品 | 对 DeepCheck 的意义 |
|----------|------|--------------------|
| 上游平台 | 长桥 OpenAPI（行情/财报/交易 API） | 数据供应商，合作关系而非竞争 |
| 同层产品 | 长桥 AI Agents / 同花顺 AI / Futu AI | 同赛道，需差异化定位 |
| 降维威胁 | 长桥 AI Skill + MCP 生态（38+ 标准化技能） | ⚠️ 最大平台风险：若长桥将"年报 RAG"做成标准 Skill，DeepCheck 独特价值被吞噬 |
| 通用替代 | ChatGPT + 联网插件 | 泛化 AI 的金融场景渗透 |

**核心差异矩阵：**

| 维度 | 长桥 AI Agents（C端） | 长桥 Skill + MCP（开发者端） | ChatGPT | DeepCheck |
|------|----------------------|----------------------------|---------|----------|
| 目标用户 | App 内散户 | 开发者 / AI 产品 | 通用用户 | 有研究需求的个人投资者 |
| 数据深度 | 结构化行情 + 财报 | 38 个标准化查询技能 | 训练截止日期内的知识 | 行情 + 财报 + **年报 RAG 语义检索** |
| 分析深度 | 持仓日报、市场摘要 | 单点查询（无综合判断） | 泛化回答，无法保证数值准确 | 多工具协同 + **端到端投研结论** |
| 对话状态 | 有限记忆 | 无状态 | 有记忆但无金融工具链 | **多轮对话记忆**（记住讨论标的） |
| 来源可追溯 | 部分标注 | API 返回值 | ❌ 无来源标注 | ✅ 每条数据强制标注来源 |
| 核心壁垒 | 交易闭环场景优势 | 开发者生态规模效应 | 通用智能 + 用户基数 | **非结构化文档深度理解** |

**DeepCheck 的差异化定位：** 长桥 AI 擅长"结构化数据的标准化查询"，DeepCheck 聚焦"非结构化年报的深度语义理解 + 端到端综合研判"——这是长桥 38 个 Skill 中尚未覆盖的能力。

## 二、用户画像与核心场景

### 2.1 目标用户

```
主要用户：有 1-3 年投资经验的个人投资者
核心需求："我想知道这只股票现在能不能买"
行为特征：会用基础财务指标，但不看原始年报，每天花 30 分钟做决策
```

### 2.2 核心用户场景（3 个）

**场景 A：快速体检**
```
用户："苹果现在能不能买？"
Agent：→ 当前价 $291.58（+0.35%）+ Q2 营收 $1111.8亿（+16.6%）
      → PE 36.83（高于历史中位数 34.28，估值偏高）
      → Cook 卸任 + WWDC AI 大考，事件面不确定
      → 综合建议：观望。等 WWDC 结果再看。
```

**场景 B：风险排查**
```
用户："苹果面临什么风险？"
Agent：→ 搜索 10-K Item 1A → 汇总 6 类风险 → 标注来源页码
```

**场景 C：跨市场对比**
```
用户："腾讯和苹果谁的利润率高？"
Agent：→ 腾讯净利率 31.72% vs 苹果 26.60% → 腾讯更高
      → 但苹果 ROE 121.5% vs 腾讯 8.4%，苹果股东回报效率更高
```

## 三、MVP 功能范围

### 3.1 已实现（MVP v1.0）

| 模块 | 功能 | 技术实现 |
|------|------|---------|
| 行情查询 | 实时股价、涨跌幅、成交量 | Longbridge Quote API |
| 财报查询 | 营收、净利润、EPS、毛利率、净利率、ROE | Longbridge Fundamental API |

| 防幻觉 | "无数据时明确说暂无数据" | Prompt 约束 |

## 四、成功指标（KPI）

| 指标 | 定义 | MVP 现状 | V1.0 目标 |
|------|------|---------|----------|
| 工具路由准确率 | 正确选择工具的查询比例 | 100%（20/20） | ≥ 95% |
| 年报覆盖数 | 已入库 10-K 股票数量 | 6 只（美股 TOP6） | 50+ 只（港股+美股） |
| 数值准确率 | 回答中的数值与 API 返回值一致 | ~80% | ≥ 95% |
| RAG 命中率 | 检索结果包含问题答案的比例 | 70%（7/10，BGE升级后） | ≥ 85% |
| 来源引用率 | 回答中标注来源的比例 | 90% | ≥ 95% |
| 平均响应时间 | 从提问到完整回答 | ~8s | < 5s |

## 五、竞品与风险

### 5.1 竞争格局

**直接竞争：**

- **长桥 AI Agents**（[longbridge.com/ai/agents](https://longbridge.com/zh-CN/ai/agents/explore)）：整合在交易 App 内，拥有场景闭环优势（查完即买）；但年报 RAG 语义检索能力有限，侧重结构化数据展示
- **长桥 AI Skill + MCP**（[open.longbridge.com](https://open.longbridge.com/docs)）：38+ 标准化技能 + MCP 协议接入 Claude/Cursor 等 AI 助手；覆盖面广但为单点查询工具，缺乏端到端综合研判能力

**间接竞争：**

- **同花顺 AI / Futu AI**：各自券商生态内的 AI 功能，以 A 股 / 港美股为主，封闭生态
- **ChatGPT + 联网**：通用 AI 的金融渗透，数据非实时且无来源追溯
- **Bloomberg GPT**：金融领域 fine-tune，专业但不可定制、高成本
- **机构自研**：高盛、桥水等自研投研工具，不对外开放

**DeepCheck 的护城河：**

| 护城河维度 | 具体表现 |
|-----------|----------|
| 非结构化理解 | 200+ 页年报的语义问答（"Apple 面临什么地缘政治风险？"），长桥 Skill 无此能力 |
| 综合研判 | 多工具协同出结论（行情 + 财报 + 年报 → "能不能买"），而非返回原始数据 |
| 来源强约束 | 架构级防幻觉（工具返回值模板 + Prompt 约束），而非仅靠模型能力 |

**平台风险应对：** 若长桥将年报 RAG 纳入 Skill 生态，DeepCheck 需向"多源年报覆盖 + 个性化研究记忆 + 投资组合级分析"方向进化，保持差异化

### 5.2 核心风险

| 风险 | 缓解策略 |
|------|---------|
| LLM 编造数据（幻觉） | 评测体系 + 来源引用 + Prompt 约束 + 检索保真度指标 |
| API 稳定性（Longbridge 偶发空返回） | 重试机制 + 降级策略（"暂时无法获取"） |
| RAG 检索质量不足 | ✅ 已升级 BGE-large-zh-v1.5（命中率 50%→70%），后续考虑混合检索 |
| 港股/美股数据合规 | 仅展示公开行情和财报数据 |
