## 一、产品背景与定位

### 1.1 为什么要做这个 Agent

个人投资者做投研决策时面临三个痛点：

| 痛点 | 现状 | Agent 解法 |
|------|------|-----------|
| 数据分散 | 行情看一个 App，财报查一个网站，新闻刷另一个 | 一次对话，Agent 自动跨数据源聚合 |
| 认知门槛高 | PE/PB/ROE 指标需要专业知识才能正确解读 | Agent 用自然语言解释指标含义 + 横向对比 |
| 信息过载 | 10-K 年报 200+ 页，核心风险埋在第 30 页 | RAG 检索 + LLM 摘要，10 秒出结论 |

### 1.2 产品定位

> DeepCheck 是一个基于长桥 OpenAPI 构建的 AI 投研助手，核心能力是**非结构化年报（10-K）的深度语义检索**。通过自然语言对话，自动调用实时行情、财报数据和年报 RAG 检索，输出带来源引用的端到端投研简报。
>
> 项目目标：在长桥现有数据生态上，验证"年报语义检索 + 多工具协同"这一产品方向的可行性。

### 1.3 长桥 AI 生态分析

目前，长桥开发平台 API 能力已覆盖 9 大模块：

| 模块 | 覆盖内容 | DeepCheck 使用情况 |
|------|---------|-------------------|
| Quote（报价） | 实时/历史报价、Stocks、Options、Warrants、Analytics、Watchlist、WebSocket 推送 | ✅ 已用（实时行情） |
| Fundamental（基本面） | 财务报告、分析师评级、**估值**、**估值历史**、**行业估值**、股息、基金持仓 | ⚠️ 仅用损益表（大量估值数据未开发） |
| Market（市场） | 市场状态、财经日历 | ❌ 未用 |
| News & Contents（新闻） | 市场新闻、社区话题、Sharelist | ❌ 未用 |
| Screener（选股器） | 选股器筛选 API | ❌ 未用 |
| Trade（交易） | 订单管理、执行查询、资产、交易推送 | ❌ 未用 |
| Account（账户） | 投资组合分析、价格提醒（Alerts）、定投（DCA） | ❌ 未用 |
| CLI | 命令行工具，涵盖以上所有功能 + 量化回测 | ❌ 未用 |
| MCP | AI 助手工作流程的模型上下文协议集成 | ❌ 未用 |

同时，长桥已上线多款 C 端 AI 产品：

| 产品 | 核心能力 | 架构模式 | 数据源 | 思维链深度 |
|------|---------|---------|--------|----------|
| LongbridgeAI 主平台 | 通用投研对话，Agent 模式 + 长桥综合账户 | Tool Calling + Plan-and-Execute 混合 | 结构化 API + 新闻 + 网页搜索 + Skills | 多步推理，可展开 |
| 宏观分析助手 | 宏观经济分析 | Plan-and-Execute | 子问题拆解 → 并行多路新闻检索 | 多步推理 |
| 长桥选股器 | 智能选股 | 工具调用（仅 2 步） | Screener API（行业/市值筛选） | 无思维链，纯 API + LLM 解读 |
| 股票情绪分析 | 情绪多维判断 | Plan-and-Execute | 6 路并行新闻检索 + 网页搜索 | 8 步深度推理 |
| 股票大V说 | 多视角观点聚合 | Plan-and-Execute | 新闻 + UGC 内容 + 网页搜索 | 10+ 步推理（多空分类 + 合规声明） |
| 热点股票挖掘 | 热门机会发现 | 工具调用 | 榜单/排行数据 | 支持优雅降级 |

**产品设计共性 Pattern：**
- **统一的子问题分解框架**：情绪分析、大V说、宏观分析均使用 `Generating Sub Questions` + `Generating Keywords` → 并行检索
- **"内部 API + 网页搜索"双源模式**：内部新闻/UGC 不够时自动外部搜索兜底
- **思维链可视化**：用户可展开推理过程每一步，建立信任
- **合规声明标准化**：所有输出结尾均附 disclaimer（"PortAI 未获发投资顾问牌照..."）
- **优雅失败**：数据获取失败时主动承认限制并提供替代建议，不编造数据

> 以上分析基于实际使用长桥 AI 产品的观察，体现了长桥在 AI 投研领域的深度布局。

DeepCheck 构建在长桥开放平台之上，与现有 AI 产品形成互补：

| 层级 | 长桥现有能力 | DeepCheck 补充 |
|------|-----------|---------------|
| 数据层 | 结构化 API（行情、财报、估值、新闻） | 非结构化年报（10-K）语义索引 |
| Agent 层 | 多步推理 + 网页搜索 + 新闻检索 | 年报 RAG 检索 + 多源综合研判 |
| 产品层 | 选股器、情绪分析、大V说等垂直场景 | "这只股票能不能买"的端到端投研简报 |
| 潜在融合 | Skill + MCP 开放生态 | 年报 RAG 可封装为标准 Skill 接入生态 |

## 二、用户画像与核心场景

### 2.1 目标用户

```
主要用户：有 1-3 年投资经验的个人投资者
核心需求："我想知道这只股票现在能不能买"
行为特征：会用基础财务指标，但不看原始年报，每天花 30 分钟做决策
```

### 2.2 核心用户场景（3 个）

**场景 A：快速体检**
```
用户："苹果现在能不能买？"
Agent：→ 当前价 $291.58（+0.35%）+ Q2 营收 $1111.8亿（+16.6%）
      → PE 36.83（高于历史中位数 34.28，估值偏高）
      → Cook 卸任 + WWDC AI 大考，事件面不确定
      → 综合建议：观望。等 WWDC 结果再看。
```

**场景 B：风险排查**
```
用户："苹果面临什么风险？"
Agent：→ 搜索 10-K Item 1A → 汇总 6 类风险 → 标注来源页码
```

**场景 C：跨市场对比**
```
用户："腾讯和苹果谁的利润率高？"
Agent：→ 腾讯净利率 31.72% vs 苹果 26.60% → 腾讯更高
      → 但苹果 ROE 121.5% vs 腾讯 8.4%，苹果股东回报效率更高
```

## 五、市场分析与风险

### 5.1 行业格局

**券商 AI 投研产品：**

- **长桥 AI**：成熟的 Multi-Tool Agent 平台 + 垂直 Agent 矩阵，覆盖结构化数据查询、新闻资讯、智能选股、情绪分析等场景，拥有交易闭环优势
- **同花顺 AI / Futu AI**：各自券商生态内的 AI 功能，封闭生态
- **Bloomberg GPT**：金融领域 fine-tune，专业但高成本、不可定制

**通用 AI 渗透：**

- **ChatGPT + 联网 / Perplexity**：通用 AI 的金融场景渗透，数据非实时且无来源追溯

**行业共性特征：** 目前主流产品的数据源集中在结构化 API（行情、财报）和新闻资讯，对年报等非结构化长文档的语义检索尚处于早期阶段。

### 5.2 DeepCheck 的核心价值

| 价值维度 | 具体表现 |
|-----------|----------|
| 非结构化年报检索 | 200+ 页年报的语义问答（如"Apple 面临什么地缘政治风险？"），从 Item 1A 中精准定位 |
| 端到端综合研判 | 行情 + 财报 + 年报 → "能不能买"的综合结论，而非单一数据展示 |
| 来源强约束 | 架构级防幻觉（工具返回值模板 + Prompt 约束），每条数据可追溯 |
| 可量化评测 | 4 维度（工具路由 / 数值准确 / 来源引用 / 答案相关性）× 20 条用例，可复现验证 |

### 5.3 对长桥的潜在价值

作为基于长桥 OpenAPI 的独立项目，DeepCheck 验证了以下方向的可行性：

| 方向 | 说明 |
|------|------|
| **年报 RAG 作为新 Skill** | 年报语义检索可封装为标准化 Skill，扩展长桥 AI 的非结构化数据能力 |
| **投研深度延伸** | 现有 Agent 擅长"实时数据 + 新闻"，年报 RAG 补充"历史披露 + 风险分析"维度 |
| **评测体系参考** | 4 维度评测框架可为 Agent 质量监控提供参考 |
| **低成本 LLM 验证** | DeepSeek 方案验证了在投研场景下低成本 LLM 的可用性边界 |

### 5.4 后续演进方向

若将此能力纳入长桥生态，可沿以下路径深化：
- 多源年报覆盖（港股年报 + A 股年报 + 招股书）
- 个性化研究记忆（用户关注的标的和风险偏好）
- 投资组合级分析（持仓整体风险评估）

### 5.5 核心风险

| 风险 | 缓解策略 |
|------|----------|
| LLM 编造数据（幻觉） | 评测体系 + 来源引用 + Prompt 约束 + 检索保真度指标 |
| API 稳定性 | 重试机制 + 降级策略（"暂时无法获取"） |
| RAG 检索质量不足 | ✅ 已升级 BGE-large-zh-v1.5（命中率 50%→70%），后续考虑混合检索 |
| 港股/美股数据合规 | 仅展示公开行情和财报数据 |
